üí° COMMON TASKS
================

ADDING A NEW SERVICE:

1. Open: admin-services.html
2. Click: "Add New Service" button
3. Fill Form:
   - Name: e.g., "Mobile App for Pharmacy"
   - Category: Select from dropdown
   - Icon: Font Awesome class (e.g., fas fa-mobile-alt)
   - Description: Service details
   - Features: Feature1, Feature2, Feature3
   - Price: Amount in numbers
   - Timeline: e.g., "2-3 weeks"
   - Status: Published/Draft
4. Click: "Save Service"
5. Result: Service appears immediately on services.html


VIEWING CUSTOMER INQUIRIES:

1. Open: inquiry-dashboard.html
2. View statistics at top
3. Options:
   - Search: Use search box (name, pharmacy, email)
   - Filter by Status: Select from dropdown
   - Filter by Service: Select service type
   - View Details: Click "View Details" button
4. Update Status: Click "Update Status" button


RESPONDING TO INQUIRY:

1. Open inquiry-dashboard.html
2. Click "View Details" on inquiry
3. Review customer information
4. Click "Send Message" button
5. In Reply Panel:
   - Write subject and message
   - Select Email and/or SMS
   - Click "Send Reply"
6. Status automatically updates to "Contacted"


EXPORTING INQUIRIES:

1. Open: inquiry-dashboard.html
2. Click: "Export" button
3. File downloads as CSV
4. Open in Excel/Sheets for analysis

‚ùì FREQUENTLY ASKED QUESTIONS
=============================

Q: Where is my data stored?
A: Browser localStorage. Only on this computer in this browser.

Q: Can I access data from another device?
A: No. Each device has separate localStorage. 
   Use backend database for multi-device sync.

Q: What if I clear browser data?
A: All module data will be deleted. 
   Regular backups recommended for production.

Q: How do customers know about services?
A: Share link to services.html with customers.
   They can view services and submit inquiries.

Q: Can I edit services from customer portal?
A: No. Only admin panel (admin-services.html) can edit.

Q: How do customers track their project?
A: Create customer portal page (future enhancement).
   Currently, admin contacts them manually.

Q: Is this mobile friendly?
A: Yes. All pages are fully responsive.
   Optimized for mobile, tablet, and desktop.

Q: Can I change colors/branding?
A: Yes. Edit CSS files or use admin settings.
   Detailed customization guide included.

Q: What browsers are supported?
A: Chrome, Firefox, Safari, Edge (modern versions)
   Requires JavaScript and localStorage enabled.
